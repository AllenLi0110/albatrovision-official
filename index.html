Platform

一套海洋方案，像產品套件一樣模組化展開

AlbatroVision 不是泛用 ERP，而是把船隊狀態、邊緣影像證據、人工稽核與報表準備 放在同一個操作中心。網站呈現方式接近套件式官網，但產品內容只服務海洋作業。

Fleet Dashboard

掌握船隻狀態、GPS、網路、魚艙溫度、近期事件與 Edge Gateway 健康狀態。

Edge Evidence

接收 Jetson 上傳的錄影、事件、縮圖、片段與傳輸確認，保留來源與時間軸證據。

Human Analysis

觀察員在完整原始影片上檢視 AI 事件、建立人工標記，保留官方可編輯判讀紀錄。

EM Reports

把時間線中的人工標記轉成電子觀察員報告候選資料，支援 CSV、XLSX 與列印輸出。

Workflow

海洋證據進來後，判讀流程要一路走到底

目前 MVP 的核心不是堆功能，而是驗證一條完整工作流：邊緣端產生證據、AlbatroVision 收斂資料、觀察員在同一條時間線上完成判讀，再準備電子觀察員報告。

  1. 01

    Edge Gateway 上傳證據

    錄影 metadata、來源影片、AI 事件、偵測框、縮圖與傳輸狀態都以 gateway 身分寫入。

  2. 02

    船隊狀態與影像合流

    Dashboard 把船舶、GPS、溫度、Camera lane 與事件摘要收斂成可掃描的操作畫面。

  3. 03

    觀察員保留官方判讀

    AI 事件是原始證據；人工標記是正式紀錄，兩者並排顯示但不互相覆寫。

  4. 04

    輸出報告候選資料

    報表工作區對齊 PM 電子觀察員欄位，讓目前可用資料和缺漏值都清楚呈現。

Current MVP Scope

現階段聚焦：海洋電子觀察員與船隊監控

AlbatroVision 現階段是 mock-first、本地可展示的產品原型，重點是驗證海洋影像稽核流程。 產品敘事保留未來延展性，但不把 roadmap 包裝成已交付功能。

現在可展示

  • 登入、RBAC、船隻切換與 vessel-scoped authorization
  • Dashboard、Video Review、Human Analysis 與 Edge Monitoring
  • Jetson Edge Gateway heartbeat、recording、event 與 source-video ingest
  • 電子觀察員欄位 catalog、報告候選與 CSV / XLSX / print output

現在不宣稱

  • 正式雲端部署、SSO、production identity 或 billing
  • 真實 RTSP / NVR / ONVIF 串流與正式 AI 推論
  • 第三方漁業系統整合或正式法規申報提交
  • 完整 manager comment 與 durable report lifecycle

可討論下一步

  • 船隊規模、資料保留、部署假設與 pilot 範圍
  • 正式 identity、source-video storage 與 report lifecycle
  • 目標市場的報告格式、欄位語系與作業術語
  • 合作夥伴 demo、投資人簡報或客戶 walkthrough
